V3 · 成体进化大版本 · REVIEW

纪元进化树 — 成体之后，旅程才刚到一半

现在的进化在成体（Lv11 / 第21天）就定格了，往后只有数字在涨、形态不再变 → 这是「Lv20 之后没动力」的根因。本版沿着已有路线再往后长两段（成熟体 / 觉醒体），正好落在你早就埋好、却还没有真实形态的两个里程碑上。纪元体本版先不做、作为预留，给以后留足升级空间。

🎯 本版目标：把养成的「可见进化」从 5 段拉到 7 段（成熟体 Lv15 / 觉醒体 Lv20），覆盖到 第 32+ 天。让一个每天上 1–2 次的玩家，每隔约一周就能亲眼看到宠物再变一次形态，并在 Lv20 亲手做一次新的成长抉择（三气觉醒）。同时掌机画框随进化适度放大、但留出上限余量——不一次做到头，纪元体及以后还能继续长。
现状基线：成体 = 终点；Lv15「羁绊渐深」/ Lv20「本命觉醒」两个称号早已存在但没有任何形态变化 → 本版给它们配上真实的身体进化。纪元体「纪元守护者」的形态留作下一版。

1

为什么要做（结合代码现状）

痛点：「宠物只变形一次还是有些单调」「玩到 20 级以后缺乏登录动力，不知道为什么要上线」「我的宠物好像升到 20 级就满级了」。

现状（代码）：web/src/data/stage-table.ts 的 STAGES 终止于 adult（order:4 / expReq:1100 / minDays:21），MAX_STAGE_V1 = "adult"。等级本身无上限（levels.ts 由 EXP 派生），但成体之后身体不再变、roadmap 也只剩数字 → 玩家把「形态不变」误读成「满级了」。

  • 已有 titles.ts 里 lv15 羁绊渐深 / awaken 本命觉醒(Lv20) / lv30 纪元守护者 三个里程碑已埋点，awaken 甚至已标 awakening:true 并在文案里承诺「点亮星辉光环」——但从没有对应的身体进化。
  • 已有 美术引擎 scripts/gen-art.mjs 用 NODE = {egg:0,baby:1,child:2,teen:3,adult:4} + SCALE = [0,0.74,0.86,0.96,1.06] 逐阶段程序化绘制，每只怪的 draw(variant,node,mood) 已支持「按 node 加特征」。本版加 2 个 node（5/6），并预留 node 7 给纪元体，不需要逐帧手绘。
  • 已有 少年期 4 形态分叉（真形 + 丰裕/敏捷/守护）已经跑通玩家亲手选形的交互（POST /api/pet/evolve）。成体后的「三气觉醒」可以复用这套分叉 UI。
💡 核心思路：不新增物种、不推翻已选形态。沿着玩家已经养出来的那只继续往上「飞升」两段（成熟体 → 觉醒体），每段都是真实可见的身体变化 + 一次里程碑庆祝 + 一份专属奖励，并在觉醒插入一次新的玩家抉择（三气）。纪元体作为预留形态，等本版稳定、美术与画框都还有余量时再开——这就是「不一次做到头」。

3

两段成体进化 — 各自的「事件感」

两段都不是简单放大像素，每段都有明确主题、一次专属变化、一次庆祝、一份只此一档能拿的奖励，让「又变了一次」成为持续上号的诱因。

① 成熟体 · 特化定型（Lv15 / 第26天）

  • 主题：少年期你选的那条分支，到这里真正「定型」——本命特征被放大锐化（樱桃绽成花、卷须更灵动、方头亮起像素纹……见 §7）。
  • 变化：体型 SCALE 1.06→1.12、姿态更挺、加一处签名特征；画框同步放大一档（见 §4）。
  • 奖励：解锁专属头饰 antlers 鹿角枝(Lv16) 的同档位伙伴件 + 称号「羁绊渐深」点亮。
  • 庆祝：复用现有升级解锁庆祝 modal（「✨ 它定型了！」），一次性。

② 觉醒体 · 本命觉醒（Lv20 / 第32天）大事件 · 本版终点

  • 主题：兑现 titles.ts 早就承诺的「点亮星辉光环」。这是本版最大的一次视觉跃迁：身后浮现内置柔光环（不占装饰槽）、额前点亮一颗星辉、姿态站定。
  • 玩家抉择：在这里弹出三气觉醒选择（见 §5）——焰 / 霜 / 辉，决定觉醒体的整体气质与光色（将来也会被纪元体继承）。这是少年分叉之后第二次亲手塑形。
  • 变化：体型 SCALE 1.12→1.18；画框再放大一档，但仍留有上限余量（见 §4）。
  • 奖励：解锁专属光环 aura_star 星辉环(Lv20) 的「觉醒」强化版 + 称号「本命觉醒」（headline）。
  • 庆祝：升级为整屏觉醒动画（淡入光环 + 星辉粒子，约 1.5s），区别于普通解锁 toast，强调「这是大事」。
③ 纪元体 · 纪元守护者（Lv30）— 本版预留，下一版再做

按你的意见本版先不实装。骨架已为它留好：阶段表预留 order 7、美术引擎预留 node 7、画框预留到上限那一档（见 §4 的虚线预留框）、三气已设计成可被它继承。等本版稳定、且画框/美术仍有余量时再开——头顶光冠、常驻粒子、三气增幅到最盛、解锁纪元环与（可选）纪元背景。

🔒 不可逆但温柔：成体进化只升不降；即使宠物进入濒危（现有系统）也只掉状态、不掉形态。形态是「成就」，不会被惩罚机制夺走，保护玩家的长期投入感。
4

掌机画框随进化适度放大（留足未来空间）

你的诉求：升级以后模拟掌机的屏幕画框可以适量增大一点；但不要一次做到头，要给以后的进化留空间。

现状（代码）：§1（V2）已把舞台固定成一块 LCD，画框约 252rpx、宠物 bound 钳到 ≤229rpx。画框大小不随阶段变化 → 越到后面、宠物越大只是更贴边，「世界」没跟着它一起长。

设计：画框与宠物一起长，比例恒定 ~85%、上限留白

  • 同步放大：每进一阶，画框 + 宠物一起放大，但两者比例恒定在 ~85%（永不溢出、永不顶满）。「它长大了」被「精灵更大」+「掌机屏幕更大」两层一起表达，体感更强。
  • 小步走、留上限：成体 252 → 成熟体 268(+6%) → 觉醒体 284(+13%)；硬上限预留到 ~312rpx（仍在手机机身边框内）。本版只用到 284，顶部那截留白专门留给以后的纪元体等新阶段——这就是「不一次做到头」。
  • 机身不变、只长屏幕：手机外壳尺寸固定，只放大里面的 LCD 取景区；放大同时相机轻微 zoom-out，过渡不突兀。

5

三气觉醒 — 终局的新分叉（art 友好）

少年期分叉是「选样貌」，觉醒期分叉是「选气质」。三气是一层调色 + 光环 overlay，叠在你已选好的那只形态上 → 不需要新画 16 套精灵，却让每只觉醒体/纪元体都长得不一样。Lv20 弹窗一次性选定，继承到纪元体。

🔥 焰 · 炽心

暖橙红调 · 火花粒子

活泼热烈的气质。光环偏暖橙，偶尔迸出小火星。适合常陪玩、互动多的玩家。

❄️ 霜 · 静月

冷蓝白调 · 雪花粒子

沉静优雅的气质。光环偏冷蓝，飘落细小雪花。适合稳定打卡、细水长流的玩家。

✨ 辉 · 暖阳

金白调 · 星点粒子

温柔明亮的气质。光环偏金白，环绕细碎星点。中庸耐看，selecting-by-default 的安全选项。

实现上：三气 = 一个 aura_qi 字段（flame|frost|radiant），渲染时给精灵叠一层 tint（轻度混色） + 切换 粒子贴图。美术引擎只需在 renderBuf 末尾按 qi 走一个 overlay 分支，复用已有 floatExtras 粒子原语。可像少年分叉一样给「推荐气」提示（按 care/互动统计），但最终由玩家定。

线上 24 只宠物的迁移安全

  • 零回退 新两段只往 adult 之后追加，不动前五段的 expReq/天数。所有现存宠物的当前阶段、等级、形态完全不变。
