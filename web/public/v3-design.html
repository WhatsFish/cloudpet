V3 · 成体进化大版本 · REVIEW

纪元进化树 — 成体之后，旅程才刚到一半

现在的进化在成体（Lv11 / 第21天）就定格了，往后只有数字在涨、形态不再变 → 这是「Lv20 之后没动力」的根因。V3 把进化沿着已有路线继续往后长三段，并让它们正好落在你早就埋好、却还没有真实形态的三个里程碑上。

🎯 本版目标：把养成的「可见进化」从 5 段拉到 8 段，覆盖到 第 50+ 天 / Lv30。让一个每天上 1–2 次的玩家，每隔约一周就能亲眼看到宠物再变一次形态，并在 Lv20 亲手做一次新的成长抉择（三气觉醒），把「等级数字没意义了」彻底解决。
现状基线：成体 = 终点；Lv15「羁绊渐深」/ Lv20「本命觉醒」/ Lv30「纪元守护者」三个称号早已存在但没有任何形态变化 → V3 给它们配上真实的身体进化。

1

为什么要做（结合代码现状）

痛点：「宠物只变形一次还是有些单调」「玩到 20 级以后缺乏登录动力，不知道为什么要上线」「我的宠物好像升到 20 级就满级了」。

现状（代码）：web/src/data/stage-table.ts 的 STAGES 终止于 adult（order:4 / expReq:1100 / minDays:21），MAX_STAGE_V1 = "adult"。等级本身无上限（levels.ts 由 EXP 派生），但成体之后身体不再变、roadmap 也只剩数字 → 玩家把「形态不变」误读成「满级了」。

  • 已有 titles.ts 里 lv15 羁绊渐深 / awaken 本命觉醒(Lv20) / lv30 纪元守护者 三个里程碑已埋点，awaken 甚至已标 awakening:true 并在文案里承诺「点亮星辉光环」——但从没有对应的身体进化。
  • 已有 美术引擎 scripts/gen-art.mjs 用 NODE = {egg:0,baby:1,child:2,teen:3,adult:4} + SCALE = [0,0.74,0.86,0.96,1.06] 逐阶段程序化绘制，每只怪的 draw(variant,node,mood) 已支持「按 node 加特征」。往后加 3 个 node 是这套引擎天然能扩的，不需要逐帧手绘。
  • 已有 少年期 4 形态分叉（真形 + 丰裕/敏捷/守护）已经跑通玩家亲手选形的交互（POST /api/pet/evolve）。成体后的「三气觉醒」可以复用这套分叉 UI。
💡 核心思路：不新增物种、不推翻已选形态。沿着玩家已经养出来的那只继续往上「飞升」三段，每段都是真实可见的身体变化 + 一次里程碑庆祝 + 一份专属奖励，并在中段插入一次新的玩家抉择（三气），把「亲手决定它长成什么样」的爽点从少年期延续到终局。

3

三段成体进化 — 各自的「事件感」

三段不是简单放大像素，每段都有明确的主题、一次专属变化、一次庆祝、一份只此一档能拿的奖励，让「又变了一次」成为持续上号的诱因。

① 成熟体 · 特化定型（Lv15 / 第26天）

  • 主题：少年期你选的那条分支，到这里真正「定型」——本命特征被放大锐化（樱桃绽成花、卷须更灵动、方头亮起像素纹……见 §6）。
  • 变化：体型 SCALE 1.06→1.14、姿态更挺、加一处签名特征。
  • 奖励：解锁专属头饰 antlers 鹿角枝(Lv16) 的同档位伙伴件 + 称号「羁绊渐深」点亮。
  • 庆祝：复用现有升级解锁庆祝 modal（「✨ 它定型了！」），一次性。

② 觉醒体 · 本命觉醒（Lv20 / 第32天）大事件

  • 主题：兑现 titles.ts 早就承诺的「点亮星辉光环」。这是整条线最大的一次视觉跃迁：身后浮现内置柔光环（不占装饰槽）、额前点亮一颗星辉、姿态站定。
  • 玩家抉择：在这里弹出三气觉醒选择（见 §4）——焰 / 霜 / 辉，决定觉醒体与纪元体的整体气质与光色。这是少年分叉之后第二次、也是终局前最后一次亲手塑形。
  • 奖励：解锁专属光环 aura_star 星辉环(Lv20) 的「觉醒」强化版 + 称号「本命觉醒」（headline）。
  • 庆祝：升级为整屏觉醒动画（淡入光环 + 星辉粒子，约 1.5s），区别于普通解锁 toast，强调「这是大事」。

③ 纪元体 · 纪元守护者（Lv30 / 第50+天）capstone

  • 主题：旅程的封顶形态。头顶光冠、持续轻粒子、三气之力增幅到最盛——是「我陪它走完了 50 天」的勋章。
  • 变化：体型 SCALE →1.26（仍受 §1 取景框 ≤229rpx 钳制，绝不溢出）、光冠 + 常驻粒子 FX。
  • 奖励：解锁全游最高光环 aura_tide 潮汐环(Lv28)→纪元环 + 称号「纪元守护者」+ 可选的纪元专属背景（见 §7 新槽位）。
  • 庆祝：纪元加冕动画 + 永久在主页显示「纪元 · 第N天」徽记。
🔒 不可逆但温柔：三段进化只升不降；即使宠物进入濒危（§现有系统）也只掉状态、不掉形态。形态是「成就」，不会被惩罚机制夺走，保护玩家的长期投入感。
4

三气觉醒 — 终局的新分叉（art 友好）

少年期分叉是「选样貌」，觉醒期分叉是「选气质」。三气是一层调色 + 光环 overlay，叠在你已选好的那只形态上 → 不需要新画 16 套精灵，却让每只觉醒体/纪元体都长得不一样。Lv20 弹窗一次性选定，继承到纪元体。

🔥 焰 · 炽心

暖橙红调 · 火花粒子

活泼热烈的气质。光环偏暖橙，偶尔迸出小火星。适合常陪玩、互动多的玩家。

❄️ 霜 · 静月

冷蓝白调 · 雪花粒子

沉静优雅的气质。光环偏冷蓝，飘落细小雪花。适合稳定打卡、细水长流的玩家。

✨ 辉 · 暖阳

金白调 · 星点粒子

温柔明亮的气质。光环偏金白，环绕细碎星点。中庸耐看，selecting-by-default 的安全选项。

实现上：三气 = 一个 aura_qi 字段（flame|frost|radiant），渲染时给精灵叠一层 tint（轻度混色） + 切换 粒子贴图。美术引擎只需在 renderBuf 末尾按 qi 走一个 overlay 分支，复用已有 floatExtras 粒子原语。可像少年分叉一样给「推荐气」提示（按 care/互动统计），但最终由玩家定。

进化门槛与时机（沿用现有三重门）

新三段照搬 stage-table.ts 的设计哲学：EXP 门 ∧ 真实天数门 ∧ 羁绊门三者同时满足才进化（不能靠单次刷 EXP 抢阶段）。觉醒/纪元再加一道「曾达成连登」软门，把「持续上号」直接写进进化条件——这是回答「为什么要上线」的硬钩子。天数按新经验曲线（gap(n)=round(90+20n+1.1n²)）反推。

  • 天数门为主、EXP 门贴边：和现有阶段一样，正常照顾下 EXP 门≈天数门同时到达，避免「明明 Lv 够了却卡住」的困惑。
  • 「曾达成」软门用高水位：读已有的 max_streak（一生最高连登），掉签也不会回退 → 鼓励连登但不惩罚偶尔断签。觉醒只要历史上连过 7 天、纪元要连过 30 天（直接服务「连续登录奖励」这条诉求）。
  • 羁绊门递增：bondGate 400→550→700→850，并作为进化后的羁绊保底（不跌破），保护长期玩家。
  • cap 维持 100：活力/饱食等实时数值仍是 0–100，新阶段不做数值膨胀，奖励集中在形态/称号/光环/背景，避免破坏现有 UI 与平衡。
⏱️ 节奏校准：成体(21)→成熟(26)→觉醒(32)→纪元(50)，间隔约 5 / 6 / 18 天。前两段贴着「每周一变」给短期目标；纪元体故意拉长成终极长线目标，配合 §8 中间穿插的称号/光环解锁，把第 32–50 天的空窗填满。

50 天体感曲线 — 每周都有盼头

把进化、称号、光环、连登里程碑铺在同一条时间轴上，确认没有「空窗周」。

第1周
蛋→幼体→幼年，学会照顾循环
第2周
少年分叉（亲手选形）→ 接近成体
第3周
成体 Lv11 → 成熟体 Lv15 特化定型
第4–5周
觉醒体 Lv20 三气抉择（大事件）+ 中途 Lv16/Lv24 头饰、Lv28 光环
第6–7周+
冲刺 纪元体 Lv30 封顶 + 连登 30/60/100 里程碑
  • 短期钩子（<1周可达）：每日连登礼、单级解锁 chip、成熟体特征。
  • 中期钩子（1–2周）：觉醒大事件 + 三气抉择 = 第 4 周的高潮，正好接住「Lv20 没动力」的断点。
  • 长期钩子（终局）：纪元体 + 纪元背景 + 连登百日，给重度玩家一个值得炫耀的终点。

线上 24 只宠物的迁移安全

  • 零回退 新三段只往 adult 之后追加，不动前五段的 expReq/天数。所有现存宠物的当前阶段、等级、形态完全不变。
  • 自然衔接 已经 Lv15+/Lv20+ 的宠物（如你的大熊 Lv20）在下一次 tick 时，会按新表自动补进化到它等级/天数/羁绊已满足的最高新阶段——并触发一次（仅一次）觉醒庆祝；觉醒分叉则在首次进入觉醒体时弹一次三气选择（在补进化场景给个温和默认 = 辉，可改）。
  • 幂等 沿用 migration_marker 模式；aura_qi 默认 NULL，渲染时 NULL 视作「未觉醒/无气」。
  • 部署 后端：docker compose up -d --build --force-recreate web + 跑列迁移；客户端：你在微信开发者工具上传新版本后对真实用户生效。
